# Project: RecipeShare

A recipe sharing web app where users can browse, search, and submit recipes.

## What I Want

A good-looking recipe website served from FastAPI with SQLite. Server-rendered
HTML with Jinja2 templates — no JavaScript frameworks. It should feel like a
real food blog, not a developer demo.

Users should be able to:
- Browse recipes by category (Breakfast, Lunch, Dinner, Dessert, Snack)
- Search by ingredient or recipe name
- Submit new recipes through a form
- See a homepage with the most recent recipes

Design should be warm earth tones (sage green, cream, terracotta), responsive,
clean typography with a serif font for headings.

Seed the database with 20 realistic recipes so it looks populated from day one.

## Stack
- FastAPI + Jinja2 + SQLite
- Pure HTML/CSS (no React, no JS frameworks)
- Deploy via ItsAlive when done
